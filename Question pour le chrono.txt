Questions à se poser pour la réalisation du chronometre:

a) Fonctionnalités de base

Quelles fonctionnalités doit avoir le chronomètre ?

Démarrer, arrêter, remettre à zéro ?

Temps en heures, minutes, secondes, millisecondes ?

Option “pause” ?

Comment l’utilisateur interagit-il avec le chronomètre ?

Boutons clairs : Start / Stop / Reset ?

Touche clavier ou uniquement clics ?

b) Interface (HTML & CSS)

Quelle structure HTML pour le chronomètre ?

<div> pour afficher le temps, <button> pour les actions.

Quel style CSS pour le rendre lisible et agréable ?

Couleurs, taille de police, animations éventuelles.

Le chronomètre doit-il être responsive (adapté aux mobiles) ?

c) Comportement (JavaScript)

Comment gérer le temps et les intervalles ?

setInterval ou requestAnimationFrame ?

Comment mettre à jour l’affichage en temps réel ?

Comment gérer les erreurs ou bugs (par ex. plusieurs clics sur Start) ?

Comment arrêter proprement le chronomètre pour éviter des calculs inutiles ?

d) Extensibilité (optionnel)

Peut-on ajouter des fonctionnalités supplémentaires ?

Compte à rebours (timer)

Tours (laps)

Sauvegarde du temps dans le navigateur
